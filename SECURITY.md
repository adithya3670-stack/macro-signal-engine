# Security Policy

Thank you for helping keep Macro Signal Engine secure.

## Supported Versions

Security fixes are provided for the latest `main` branch only.

| Version/Branch | Supported |
| --- | --- |
| `main` | Yes |
| Historical branches/tags | No |
| Forks/custom deployments | No |

## Reporting a Vulnerability

Preferred channel (private):

1. Open a GitHub Security Advisory draft for this repository.
2. Include reproduction steps, impact, and any proof-of-concept details.

Fallback channel (private):

- Email: `adithya3670@gmail.com`

Please do not open public issues for undisclosed vulnerabilities.

## Response Targets

These are best-effort targets, not guarantees:

- Acknowledgment: within 3 business days
- Initial triage update: within 7 business days
- Follow-up status cadence: every 14 days while open
- Remediation window: best-effort fix or mitigation within 90 days for supported code

## Scope and Non-Goals

Out of scope for this policy:

- Issues in unsupported branches/tags
- Vulnerabilities introduced by downstream forks or custom infrastructure
- Findings that require access to third-party services outside repository control

## Coordinated Disclosure

Please allow time for triage and remediation before public disclosure.
After a fix is available, maintainers will coordinate disclosure timing and
credit where appropriate.
